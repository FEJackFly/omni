# 测绘工具 (Measure Tool) 开发维护文档

## 1. 功能概述

测绘工具集成了基于 MapLibre GL JS 的地理空间测量功能，支持以下模式：

- **距离测量 (Distance)**：支持多点连续测距，每个节点实时显示分段累计距离。
- **面积测量 (Area)**：支持多边形面积计算，面积结果自动定位在几何中心，并提供实时闭合轮廓预览。

## 2. 核心文件结构

- `src/lib/components/measure/MeasureTool.svelte`: UI 组件，负责地图交互事件监听、图层渲染管理和 Svelte 5 响应式逻辑。
- `src/lib/components/measure/measure.svelte.ts`: 工具函数库，包含经纬度格式化、GeoJSON 构建、距离/面积换算以及几何中心计算算法。

## 3. 技术实现要点

### 3.1 响应式模型 (Svelte 5)

- 使用 `$state` 管理坐标点集 (`points`) 和测量状态。
- 使用 `$effect` 配合 `untrack` 隔离地图图层的初始化与销毁逻辑，防止因坐标更新导致的图层反复创建导致的性能问题和渲染闪烁。

### 3.2 地图渲染

- **数据源**: 使用单个 GeoJSON 数据源 `measure-source` 容纳点、线、面要素。
- **图层**: 包含填充层 (`measure-fill`)、线条层 (`measure-line`) 和圆点层 (`measure-points`)。
- **弹窗管理**: 使用 `maplibregl.Popup` 数组管理结果标注。为了保证视觉 premium，已移除默认箭头（Tip）并优化了位移偏移。

### 3.3 算法说明

- **距离计算**: 调用 `@turf/length`。
- **面积计算**: 调用 `@turf/area`。
- **几何中心**: 为了降低外部依赖冲突风险，采用手动计算平均中心点算法（Mean Center），足以满足 UI 标注定位需求。

## 4. 交互说明

1.  **左键单击**: 采集坐标点。
2.  **右键单击或 Escape**: 撤销上一个采集点。
3.  **鼠标移动**: 实时预览下一节点的延长线或闭合面积。
4.  **双击或切换模式**: 完成当前测量，锁定结果。

## 5. 注意事项

- **单位换算**: 距离自动在 `m` 和 `km` 间切换（阈值 为 1000m），面积自动在 `m²` 和 `km²` 间切换（阈值 为 1km²）。精度通过 `toFixed(2)` 统一保留两位小数。
- **CSS 作用域**: 弹窗样式通过 `:global(.measure-popup)` 进行全局修饰，以确保 `maplibregl` 动态生成的 DOM 能正确应用样式。
